# 🔐 Security Policy

## Filosofía de Seguridad

XMLShield está diseñado con seguridad desde su núcleo:
- 100% local - Sin envío de datos
- Cero dependencias - Sin código de terceros  
- Auditable - ~1500 líneas de código visible en un solo archivo

## Reportar Vulnerabilidades

Si encuentras una vulnerabilidad de seguridad:

**Email:** Disponible en [luisfemojica.com](https://luisfemojica.com)

Por favor NO abras un Issue público para problemas de seguridad.

## Alcance de Seguridad

### En Alcance ✅
- Vulnerabilidades XSS en el resaltado de sintaxis
- Ataques de inyección via XML malicioso
- Bugs que comprometan privacidad del usuario

### Fuera de Alcance ❌
- Vulnerabilidades del navegador mismo
- Extensiones de navegador maliciosas

## Historial

No se han reportado vulnerabilidades hasta la fecha.
